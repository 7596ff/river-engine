Modify parse_emoji() function signature from:
```rust
fn parse_emoji(emoji: &str) -> RequestReactionType<'_>
```
to:
```rust
fn parse_emoji(emoji: &str) -> Result<RequestReactionType<'_>, DiscordAdapterError>
```

Replace the current if/let logic with explicit error returns:

1. If emoji format is not `<:name:id>` (parts.len() < 3), return:
   `Err(DiscordAdapterError::InvalidEmojiFormat(format!("Custom emoji must have format <:name:id>, got: {}", emoji)))`

2. If ID parsing fails (parts[2].parse::<u64>() fails), return:
   `Err(DiscordAdapterError::InvalidEmojiId(format!("Invalid emoji ID: {} (not a number)", parts[2])))`

3. For successful custom emoji, wrap in Ok()

4. For Unicode emoji (no angle brackets), wrap in Ok()

Update all call sites of parse_emoji() to handle Result using `?` operator or explicit match. Search for `parse_emoji(` to find all usages.

Add `use crate::error::DiscordAdapterError;` at top of file.

DO NOT use unwrap() or expect() on the Result - propagate errors to callers using `?`.
